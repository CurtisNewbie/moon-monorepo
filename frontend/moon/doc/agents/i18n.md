# I18n Style Guide For Agents

All hardcoded i18n values should be changed to use i18n.service.ts.

## Preferred Approach: Pure Pipe (PERFORMANCE OPTIMIZED)

**Use the `trl` pure pipe for all static translation keys in templates.** This is the preferred approach because it significantly improves performance by only executing once at initialization instead of on every change detection cycle.

### Pipe Syntax

```
{{'module' | trl:'key'}}
```

- `module` (before `|`): The module name - **first argument to pipe**
- `trl`: The pipe name
- `key` (after `:`): The translation key - **second argument to pipe**
- Additional arguments can be added with more `:` for interpolation

### Examples

**Simple translation:**

```html
<!-- Before -->
<h3>{{trl('loginHistory')}}</h3>

<!-- After -->
<h3>{{'access-log' | trl:'loginHistory'}}</h3>
```

**With interpolation:**

```html
<!-- Before -->
<h5>{{trl('importCashflows', 'importType', importType)}}</h5>

<!-- After -->
<h5>{{'pdf-viewer' | trl:'importCashflows':'importType':importType}}</h5>
```

**Multiple interpolation args:**

```html
<!-- Before -->
<p>{{trl('welcomeMessage', 'userName', user.name, 'date', today)}}</p>

<!-- After -->
<p>{{'user-detail' | trl:'welcomeMessage':'userName':user.name:'date':today}}</p>
```

### Component Changes

When using the pipe, you do **NOT** need to:
- Inject `I18n` service in the component
- Implement a `trl()` wrapper function in the component

Simply update the template to use the pipe syntax.

### Example: Complete Refactor

**Before (component.ts):**
```ts
import { I18n } from "../i18n.service";

@Component({...})
export class AccessLogComponent {
  constructor(public i18n: I18n) {}

  trl(k) {
    return this.i18n.trl("access-log", k);
  }
}
```

**Before (component.html):**
```html
<h3>{{trl('loginHistory')}}</h3>
```

**After (component.ts):**
```ts
@Component({...})
export class AccessLogComponent {
  // No I18n injection, no trl() function needed
}
```

**After (component.html):**
```html
<h3>{{'access-log' | trl:'loginHistory'}}</h3>
```

## Legacy Approach: Service + Wrapper Function (NOT RECOMMENDED)

**Only use this approach for dynamic translation keys that cannot be expressed as string literals in templates.**

Components that need i18n internationalization, should inject `I18n` service.

E.g.,

```ts
import { I18n } from "../i18n.service";

  constructor(
    public i18n: I18n,
    ...
  ) {}
```

Then inside each component, there should be a function that call `this.i18n.trl(..)` for translation, first arg should be name of the component, the second arg being the key of the translated term, and the remaining varargs are for interpolation.

E.g.,

```ts
  trl(k, args...) {
    return this.i18n.trl("manage-files", k, ...args)
  }
```

With `trl(...)` function implemented, inside the component template, rewrite previous hardcoded i18n values with `{{trl('...')}}`.

If you found something similar to following example, where `importType` is a var referenced, you should use interpolation.

E.g.,

Before:

```html
<h5>Import {{ importType }} Cashflows:</h5>
```

After

```html
<h5>{{trl('importChahflows', 'importType', importType)}}</h5>
```

```ts
// in translate.dic.ts
{
  importCashflows: "Import ${importType} Cashflows:",
}
```

## Guidelines for Agents

1. **Always prefer the pure pipe approach** for static translation keys
2. Only use the service + wrapper function approach for dynamic keys (e.g., keys constructed at runtime)
3. When refactoring an existing component:
   - Update template to use pipe syntax
   - Remove `I18n` service injection from constructor
   - Remove `trl()` wrapper function from component
4. If user asks you to fix all components, be sure to fix 1 component only at a time. After fixing one component, always ask user to check the modification and whether they want to continue.

Notice that in a single component template file (*.component.html), there can be multiple components in the same file, so you will need to inject `I18n` and impl `trl(...)` in each of these components if using the legacy approach.

A good example would be the `access-log.component.ts` in access-log component.
